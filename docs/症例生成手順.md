# 聴覚検査教育用シミュレーションにおける症例生成手順

本ドキュメントは、パラメータに基づいて聴覚検査用症例を生成する技術的手順を整理したものである。特許出願・技術説明の参考として利用できる。

---

## 1. 概要

聴覚検査の教育・訓練用に、**パラメータを設定し、それに基づいてオージオグラム（聴力図）および臨床情報（主訴・鼓膜所見等）を一貫した症例として生成する**手順である。

---

## 2. パラメータの種類

症例生成に用いる主なパラメータは以下のとおりである。

| パラメータ | 内容・取り得る値 | 役割 |
|------------|------------------|------|
| **疾患プロファイル** | Normal, SNHL_Age, SNHL_NoiseNotch, SNHL_Meniere, SNHL_Sudden, SNHL_Mumps, CHL_OME, CHL_AOM, CHL_Otosclerosis, CHL_OssicularDiscontinuity | 聴力パターン・ティンパノ型・臨床文面の種類を決定 |
| **性別** | Male, Female | 基準聴力（ISO 7029等）の参照および症例文の性別表記 |
| **年齢グループ** | 20s, 30s, 40s, 50s, 60s, 70s（または10代〜70代） | 基準聴力・年齢に応じた症例文の整合性 |
| **重症度** | プロファイルごとの定義（軽度・中等度等） | 気導・骨導閾値の変動範囲 |
| **患側** | 右耳／左耳／両耳 | 片側難聴時の主訴・左右別データの一貫性 |

上記のうち、**疾患プロファイル**が症例の聴力パターンと臨床情報の対応を決める中核パラメータである。

---

## 3. 症例生成の手順（ステップ）

### ステップ1：パラメータの決定

- 疾患プロファイル、性別、年齢グループ、重症度、患側を、所定の重みまたは乱数に基づいて**1症例分**決定する。
- ユーザーが明示的に指定している場合はその値を、指定がない場合はシステムが上記に従い自動決定する。

### ステップ2：オージオグラム（聴力図）の生成

1. **基準聴力の参照**  
   性別・年齢グループに応じた基準聴力データ（例：ISO 7029に準拠したデータ）を参照する。

2. **周波数ごとの閾値の算出**  
   疾患プロファイルに応じた変換ルール（感音性・伝音性・混合性のパターン、気骨導差、マスキング規則等）を適用し、各周波数における気導閾値および必要に応じて骨導閾値を算出する。

3. **閾値の制約**  
   医学的に妥当な範囲（周波数ごとの上下限、5dB刻み等）に収まるよう補正する。

4. **左右耳データの生成**  
   患側が片側の場合は、患側に上記パターンを適用し、健側は基準聴力に基づく正常パターンとする。両側の場合はプロファイルに応じた両耳データを生成する。

以上により、**1症例分のオージオグラムデータ（ターゲット閾値の集合）**を得る。

### ステップ3：ティンパノグラム型の決定

- 疾患プロファイルから、対応するティンパノグラム型（A型, As型, Ad型, B型, C型等）を**一意または確率的に**決定する。
  - 例：CHL_Otosclerosis → As型、CHL_AOM → C型、CHL_OME → B型またはC型。
- 必要に応じて、ピーク圧（daPa）やコンプライアンス等のパラメータを、プロファイルに応じた範囲（例：AOMのピーク圧を+50〜+200 daPa）で設定する。

### ステップ4：臨床情報（主訴・鼓膜所見等）の取得

1. **症例データベースの参照**  
   疾患プロファイル（またはそれに対応する疾患キー）およびティンパノグラム型をキーとして、事前に登録された症例データベース（CSV/JSON等）を検索する。

2. **該当症例の採用**  
   該当する症例が存在し、かつ主訴または鼓膜所見が登録されている場合、その症例の主訴・鼓膜所見・性別等を**当該生成症例の臨床情報として採用**する。

3. **データベースにない場合の補完**  
   該当症例が存在しない場合、または主訴・鼓膜所見が空である場合は、次のいずれかにより補完する。  
   - **外部AI（例：大規模言語モデル）の利用**：オージオグラムデータ・プロファイル・年齢等を入力とし、主訴・鼓膜所見・性別を一貫した自然文として生成する。  
   - **疾患別テンプレート**：プロファイルに対応した固定の主訴・鼓膜所見のテンプレートを適用する。  
   - **汎用フォールバック**：患側に応じた短文（例：「右耳の聞こえが悪い」）およびティンパノ型に応じた鼓膜所見の短文を適用する。

4. **一貫性の担保**  
   採用する臨床情報は、**ステップ1で決定した疾患プロファイルおよびティンパノ型と矛盾しない**ようにする（例：耳硬化症の場合は主訴に「徐々に」「家族歴」等を含め、鼓膜所見に「正常」「As型」「反射消失」等を含める）。

### ステップ5：症例の統合と出力

- ステップ2で得たオージオグラムデータ、ステップ3で得たティンパノグラム設定、ステップ4で得た主訴・鼓膜所見・性別等を**1つの症例**として統合する。
- 必要に応じて、ART（アブミ骨筋反射）設定やDPOAE設定など、同一症例として整合した検査パラメータを付与する。
- 統合された症例を、教育用シミュレーション画面での表示・保存・評価に供する形式で出力する。

---

## 4. 手順の効果

- **パラメータを変更するだけで多様な症例を生成できる**ため、教育者が症例を手作業で大量に用意する負担を軽減できる。
- **聴力図と主訴・鼓膜所見が同一のパラメータ・プロファイルに基づいて生成される**ため、症例内の一貫性を保ちやすく、学習者に矛盾の少ない症例を提供できる。
- **症例データベースを優先し、不足分をAIまたはテンプレートで補完する**構成により、既存の教材を活用しつつ、網羅性や表現の多様性を確保できる。

---
