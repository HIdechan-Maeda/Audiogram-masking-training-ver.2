# CSV症例データベース

## ファイル配置場所
このフォルダ（`public/data/`）にCSVファイルを配置してください。

## ファイル命名規則
疾患ごとにCSVファイルを作成します：
- `NORMAL.csv` - 正常聴力
- `AOM.csv` - 急性中耳炎
- `OME.csv` - 滲出性中耳炎
- `MUMPS.csv` - ムンプス難聴
- `SUDDEN.csv` - 突発性難聴
- `MENIERE.csv` - メニエール病
- `OTOSCLEROSIS.csv` - 耳硬化症
- `NOISE.csv` - 騒音性難聴
- `PRESBYCUSIS.csv` - 加齢性難聴
- `OSSICULAR_DISCONTINUITY.csv` - 耳小骨離断
- `ACOUSTIC_TRAUMA.csv` - 音響外傷

## CSVファイルの構造

### 必須カラム
- `caseId` - 症例ID（文字列または数値）
- `chiefComplaint` - 主訴（文字列）
- `hpi` - 病歴（文字列）
- `otoscopy` - 鼓膜所見（文字列）

### オプションカラム
- `age_min` - 年齢範囲（最小値、数値）
- `age_max` - 年齢範囲（最大値、数値）
- `gender` - 性別（M/F/男性/女性）
- `notes` - 備考（文字列）

### 注意事項
- **聴力データ（AC値、BC値）は含めません** - AI症例生成時に自動生成されます
- **ティンパノグラム型も含めません** - AI症例生成時に自動生成されます
- 鼓膜所見から適切なティンパノグラム型と聴力パターンが自動的に生成されます

## CSV例

```csv
caseId,chiefComplaint,hpi,otoscopy,age_min,age_max,gender
OME-01,聞き返しが増えた／テレビの音量が大きい（保護者気づき）,1か月前に上気道炎。集団保育（保育園）在籍。耳痛は明確でない。,両側鼓膜は灰白〜琥珀色で光錐不鮮明、気泡を認め、可動性低下（陰圧で陥凹）,4,4,M
```

**注意**: 
- 聴力データ（AC値、BC値）は含めません。AI症例生成時に自動生成されます。
- ティンパノグラム型も含めません。AI症例生成時に自動生成されます。

## 使用方法

1. CSVファイルをこのフォルダに配置
2. アプリ起動時に自動的に読み込まれます
3. CSVファイルがない場合は、既存のJSONファイルから読み込みます（フォールバック）

## 編集のコツ

- ExcelやGoogleスプレッドシートで編集可能
- UTF-8エンコーディングで保存してください
- カンマを含む文字列はダブルクォートで囲んでください
- 空欄は空のままにしてください（nullとして扱われます）
